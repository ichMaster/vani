Version 0.2.0 (2026-06-01)
---------------------------
Version 0, Phase 1 — Planner Skeleton and Tiers. Adds the deterministic
per-turn pipeline on top of the P0 skeleton.

- VANI-011 Pipeline contracts: PerceptionResult and TurnPlan (serializable, schema-validated).
- VANI-012 Perception step: one Haiku call classifies the message (topic/intent + confidence).
- VANI-013 Deterministic router: simple -> Haiku, deep -> Opus, with config-driven thresholds.
- VANI-014 Dispatch and post-update: perception + routing wired into the gated turn loop; route recorded.
- VANI-015 LLM failure handling: retry + filler on transient errors, honest fallback, consistent state.
- VANI-016 Prompt-prefix assembly: cached system prefix + fresh suffix (prompt caching).
- VANI-017 Per-turn telemetry: route, latencies, per-tier token usage; redacted, schema-conformant.
- VANI-018 TUI token/cost meter: per-turn + cumulative tokens and estimated cost in the status line.

Version 0.0.1 (2026-06-01)
---------------------------
Version 0, Phase 0 — Chat Skeleton (TUI). The first working build: a local,
streaming text chat with a transport-agnostic brain, behind clean contracts.

- VANI-001 Project scaffold and tooling: src/ package, pyproject (.venv, ruff, pytest), config loader, entry point.
- VANI-002 Repository interface and json_store: single state-access point with atomic JSON persistence and a cipher hook.
- VANI-003 Data contracts and versioned schemas: Session/Turn models, schema_version + migrate-on-read, JSON Schema validation.
- VANI-004 LLM client (Opus, streaming): Anthropic client behind a mockable LLMClient protocol with token usage.
- VANI-005 Transport-agnostic engine: Engine.handle_turn over the repository (no globals), the seam for the v2 API.
- VANI-006 TUI loop and transcript persistence: Textual chat adapter that reloads history and streams replies.
- VANI-007 Minimal guardrail pass: synchronous placeholder Guardian; output is gated before it reaches the UI.
- VANI-008 Placeholder canon and confidence scaffolding: a minimal identity compiled into the system prompt; Confident[T].
- VANI-009 Logging and telemetry scaffolding: structured logger and a redacting, repository-backed telemetry sink.
- VANI-010 pytest harness and headless replay: shared fixtures and a network-free replay of full conversations.
